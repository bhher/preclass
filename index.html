VS Code 사용법 (2.5시간)

목표

  • 프로젝트 폴더 열기/파일 관리에 익숙해진다
  • 터미널로 명령 실행(폴더 이동, Git 기본)한다
  • 확장(Extensions)과 설정으로 작업 속도를 올린다
  • 라이브 프리뷰(또는 Live Server)로 HTML 결과를 빠르게 확인한다

필수 기능 10가지

  1. Explorer: 폴더/파일 생성·이동
  2. Search: 전체 검색(Ctrl+Shift+F)
  3. Command Palette: 명령 검색(Ctrl+Shift+P)
  4. Terminal: 통합 터미널(Ctrl+`)
  5. Format: 자동 정렬(Shift+Alt+F)
  6. Multi-cursor: 여러 커서(Alt+Click)
  7. Rename Symbol: 변수명 일괄 변경(F2)
  8. Git: 변경사항 확인/커밋
  9. Snippets: 빠른 코드 생성
  10. Settings: 저장 시 포맷/탭 사이즈

추천 확장(선택)

  • Live Server: HTML 빠른 실행
  • Prettier: 코드 포맷터
  • ESLint: 자바스크립트 품질
  • GitLens: Git 히스토리 보기
  • Markdown All in One: README/노트 작성

실습(30–40분)

  1. 새 폴더 만들기 → VS Code로 열기
  2. index.html 생성 후 기본 HTML 작성
  3. Live Server로 브라우저에서 확인
  4. 저장할 때 포맷 적용되게 설정(Prettier)

체크리스트

  • 폴더/파일 생성·이동을 할 수 있다
  • 통합 터미널을 열고 명령을 실행할 수 있다
  • 저장 시 자동 포맷이 된다
  • 브라우저 미리보기로 빠르게 확인한다

따라하기 1) 프로젝트 폴더 만들기 & 열기

  1. 컴퓨터에 영문 폴더를 하나 만듭니다. 예: E:\dev\preclass
  2. VS Code 실행 → File > Open Folder로 폴더를 엽니다
  3. 왼쪽 Explorer에서 index.html, styles.css를 생성합니다
윈도우 팁
한글 경로/공백이 많으면 도구 설치나 실행에서 꼬일 때가 있어요. 처음엔 영문 폴더로 시작하는 게 안전합니다.

따라하기 2) 통합 터미널 기본

VS Code 안에서 명령어를 실행해 Git/Node 등을 다루게 됩니다.

# (PowerShell) 현재 폴더 확인
pwd

# 폴더 목록 보기
ls

# 상위 폴더로 이동
cd ..

따라하기 3) 저장할 때 자동 정렬(Format on Save)

초보 때는 “코드가 삐뚤어져서 틀린 것처럼 보이는 문제”를 줄이는 게 중요합니다.

간단 HTML/CSS 사용법 (2.5시간)

목표

  • HTML 뼈대(시맨틱)로 구조를 잡는다
  • CSS로 글꼴/색/여백/레이아웃을 만든다
  • Flex로 간단한 카드/네비 레이아웃을 만든다

핵심 개념

  • 시맨틱 태그: header, nav, main, section, footer
  • 박스 모델: margin/padding/border
  • 선택자: class 기반 설계
  • 레이아웃: Flex(가로정렬/간격)
  • 반응형 기초: max-width, 미디어쿼리 맛보기

실습(60–80분): 미니 소개 페이지

  1. 상단 네비(홈/소개/연락)
  2. 소개 카드 3개(관심사/목표/현재 상태)
  3. 푸터에 이메일/깃허브 링크

체크리스트

  • HTML 구조를 시맨틱으로 짤 수 있다
  • class 네이밍으로 스타일을 관리한다
  • Flex로 카드 3개를 깔끔히 배치한다
  • 기본 반응형(모바일에서 줄바꿈)이 된다
미니 과제: “내 소개 페이지”를 GitHub에 올리고, GitHub Pages로 배포(선택)해 보세요.

HTML: 뼈대(구조) 잡는 법

페이지를 “의미 있는 블록”으로 나누면, 나중에 CSS/JS를 붙이기 쉬워집니다.

<header>
  <nav>메뉴</nav>
</header>
<main>
  <section>소개</section>
  <section>프로젝트</section>
</main>
<footer>연락처</footer>

CSS: 기본 문법(선택자 → 속성)

/* h1 태그에 적용 */
h1 {
  font-size: 32px;
}

/* .card 클래스에 적용 */
.card {
  padding: 16px;
  border: 1px solid #ddd;
}

박스 모델(자주 실수)

용어 의미 자주 하는 실수
margin 바깥 여백 요소 사이 간격을 padding으로 해결하려 함
padding 안쪽 여백 배경색이 padding 영역까지 칠해지는 걸 모름
border 테두리 두께 때문에 박스가 커지는 걸 놓침
box-sizing 크기 계산 방식 기본값(content-box) 때문에 레이아웃이 삐뚤어짐
핵심 1줄
* { box-sizing: border-box; }를 넣으면 레이아웃이 훨씬 편해집니다.

Flex로 “카드 3개” 만들기 (복붙용 예제)

HTML

<section class="cards">
  <article class="card">카드 1</article>
  <article class="card">카드 2</article>
  <article class="card">카드 3</article>
</section>

CSS

.cards {
  display: flex;
  gap: 12px;
  flex-wrap: wrap;
}

.card {
  flex: 1 1 240px; /* 최소 240px, 공간에 따라 늘어남 */
  padding: 16px;
  border: 1px solid #ddd;
  border-radius: 12px;
}

깃 / 깃허브 사용법 (2.5시간)

목표

  • 버전관리의 의미(되돌리기/기록/협업)를 이해한다
  • 로컬 Git 흐름(add → commit)을 익힌다
  • GitHub로 원격 저장소(push)하고 README를 작성한다

핵심 흐름

  • Working Directory: 작업 중인 파일
  • Staging: 커밋에 포함할 변경 선택
  • Commit: 스냅샷(기록)
  • Remote(GitHub): 백업/공유

실습(60–90분): “첫 저장소” 만들기

  1. 프로젝트 폴더에서 Git 초기화
  2. .gitignore 추가(예: node_modules)
  3. 첫 커밋 만들기
  4. GitHub 저장소 생성 후 연결
  5. push로 올리기
  6. README에 프로젝트 소개/실행 방법 작성

체크리스트

  • add/commit/push의 차이를 설명할 수 있다
  • 커밋 메시지를 “무엇을/왜” 중심으로 쓴다
  • README에 설치/실행/기능을 적는다

처음 1번만 하는 설정(권장)

내 커밋에 이름/이메일이 찍히도록 설정합니다.

git --version

git config --global user.name "홍길동"
git config --global user.email "gildong@example.com"

git config --global -l
개념 정리
- commit: 내 PC에 기록 남기기
- push: GitHub에 업로드(백업/공유)
- pull: GitHub에서 내려받아 최신화

실습: 로컬 → GitHub까지 한 번에

  1. 프로젝트 폴더에서 git init
  2. .gitignore 만들고 불필요 파일 제외
  3. git add . → git commit -m "init"
  4. GitHub에서 새 저장소 생성
  5. git remote add origin ... 후 push
명령어 예시(복붙용) 
git init
git status

# 예: node 프로젝트라면 node_modules는 제외
echo node_modules/ > .gitignore

git add .
git commit -m "init: first commit"

git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main

실수했을 때(자주 쓰는 되돌리기)

상황 추천 명령 설명
스테이징 취소 git restore --staged . add 한 걸 다시 내림
파일 변경 취소 git restore . 작업 내용을 마지막 커밋 상태로
커밋 메시지 수정 git commit --amend 마지막 커밋을 고침
되돌리는 커밋 만들기 git revert <hash> 협업/공유 후에는 revert가 안전
자주 쓰는 Git 명령(치트시트)
  • git status: 현재 상태 확인
  • git add .: 변경 스테이징
  • git commit -m "메시지": 커밋
  • git log --oneline: 히스토리
  • git branch/git switch -c feat/x: 브랜치
  • git push -u origin main: 원격 업로드

노션 사용법 (1.5시간)

목표

  • 학습 노트와 과제 관리를 한 곳에서 한다
  • DB(Database)로 “할 일/학습 로그”를 관리한다
  • 프로젝트 템플릿을 만들어 반복 사용한다

필수 블록/기능

  • Heading: 문서 구조
  • To-do: 체크리스트
  • Callout: 중요 메모
  • Code: 코드 기록
  • Database: 표/보드/캘린더 뷰

실습(40–60분): 학습 대시보드 만들기

  1. “오늘 할 일(To-do)” DB 만들기
  2. “학습 로그” DB 만들기(날짜/주제/링크/회고)
  3. 보드 뷰(진행중/완료) 구성
  4. 프로젝트 템플릿: 목표/기능목록/이슈/회고 섹션 추가

체크리스트

  • DB를 만들고 속성(날짜/태그)을 추가한다
  • 같은 DB를 보드/표로 뷰 전환한다
  • 학습 로그를 매일 1줄이라도 남긴다
